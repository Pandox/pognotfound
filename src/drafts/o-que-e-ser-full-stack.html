Introdução

Essa cadeia de desenvolvimento, para mim, compreende:

Entendimento do negócio

  • Saber ouvir o cliente e entender sua necessidade
  • Possuir análise crítica para criar a melhor solução para o problema do cliente

Arquitetura técnica

  • Saber escolher a melhor tecnologia para a necessidade do cliente. Muitos desenvolvedores escolhem uma tecnologia por paixão ou modismo.
  • Escolher a melhor solução para cada camada do sistema, desde o sistema operacional, versionamento, banco de dados e sistema de cache.

Experiência do Usuário

  • Um bom desenvolvedor presa pelo cliente e de como ele se interage com o sistema.
  • Ele deve ter conhecimentos e saber usar sua experiência para sempre melhorar a experiência do usuário.
  • Deve conhecer técnicas de testes A/B, prototipação, análise de heat-maps e softwares para medição desses dados.

Lógica de negócio

  • Saber criar uma boa API que representa o modelo de negócio. A maneira como algumas APIs são desenvolvidas me arrepiam de medo.
  • Como esta API será utilizada por outras pessoas. Saber medir o throughput, controlar o acesso e manter a segurança da API é fundamental.
  • Conhecer governança de software.
  • Ter sólidos conhecimentos em orientação a objeto, programação funcionanl, banco de dados relacional e não relacional.

Interface do usuário

  • Saber montar o visual do seu sistema é fundamental.
