Smartycoffee - A cafeteira inteligente

Para construir uma API REST, precisamos saber qual problema ela atende. Neste caso fomos contratados por uma empresa que fábrica máquinas de café, aquelas que você bota a cápsula, para construir os serviços de TI para uma máquina ultra-inteligente, a Smartycoffee.

O cliente através do seu celular ou relógio inteligente poderá apertar um botão e solicitar para a Smartycoffee preparar alguma bebida, seja ela um café expresso, cappuccino ou chá verde. Basta ele (e a máquina) estar conectado na internet (WiFi) para fazer a solicitação. Caso a máquina não tenha agua, acabe a cápsula ou não tenha os copinhos descartáveis, o usuário deve receber um alerta.

Requisitos Funcionais

Solicitar bebida
  • O usuário, através do aplicativo de celular, escolhe uma bebida entre [Chá verde, Café expresso ou Cappuccino]
  • A máquina recebe o pedido e começa a preparar a bebida
  • A máquina termina de fazer o pedido e avisa o cliente que terminou
Monitoramento

A máquina deve fornecer informações para o usuário da seguinte natureza:

  • Quantidade de cápsulas restantes
  • Quantidade de copos restantes
  • Quantidade de bebidas realizadas
Comprar cápsulas

Pelo aplicativo será possível comprar novas cápsulas para a máquina.

REST API

Vamos seguir o padrão REST para nossa API e usar o protocolo HTTP. Para facilitar este tutorial, vamos ter uma URL base como https://smartycoffee.pognotfound.com.br/api .

Cada cliente que comprar uma máquina nova, receberá um ID único númerico da máquina. Por simplicidade, este ID é incremental.

O princípio básico do REST está em volta dos seus recursos. Os recursos são definidos na URL, e suas ações através dos métodos HTTP (GET, POST, PUT...).

Definindo os Recursos

Analise os requisitos funcionais e levante o maior número de substantivos que fazem parte do negócio.

Na sessão anterior destaquei alguns substantivos para facilitar. Consegue identificar todos?

Temos Máquina, Bebida, Pedido e Informações como substantivos. Esses são nossos recursos. Vamos deixar a compra de cápsulas para depois.

Cada recurso receberá uma URL única de identificação. Por exemplo: https://smartycoffee.pognotfound.com.br/api/maquinas - Representa a coleções de máquinas

Mapeando as Ações. É PUT ou POST?

Cada recurso possui diversas ações. Vamos começar com a bebida. Você pode solicitar uma nova bebida, e verificar o status dela.
